# Удаление и точечное редактирование тренировок

## Контекст и цель

gym95 сейчас позволяет только создавать тренировки (`/new_workout`) и смотреть историю (`/history`). Нет способа исправить опечатку (вес, повторы, дистанция) или удалить лишнюю запись — единственный вариант сейчас — жить с ошибкой в дневнике. Это первая из четырёх запланированных фич (после неё: заметки к тренировке, учёт личных рекордов, повтор тренировки по шаблону); каждая — отдельный спек.

**Область этого спека:** удаление тренировки целиком и точечное редактирование её полей. Не входит: смена типа тренировки, добавление/удаление отдельных упражнений внутри существующей тренировки — для этого проще удалить и создать заново.

## Точка входа

Под каждой карточкой тренировки в `/history` — ряд из двух inline-кнопок: `✏️ Изменить` и `🗑 Удалить`. Никаких отдельных команд или ручного ввода id — id тренировки (`uuid`) зашит в `callback_data` (`we:<id>:<offset>` / `wd:<id>:<offset>`), офсет нужен, чтобы после действия вернуться на ту же страницу истории.

## Удаление

1. Тап `🗑` → бот заменяет сообщение карточки на подтверждение: «Удалить тренировку от `<дата>` (`<тип>`)? Это нельзя отменить.» с кнопками `✅ Да` / `❌ Отмена`.
2. `✅` → `DELETE FROM workouts WHERE id = ? AND user_id = ?`. Проверка `user_id` обязательна — `callback_data` содержит только id тренировки, без сверки владельца можно было бы (в теории, зная чужой uuid) удалить не свою запись.
3. В схеме БД уже настроен `ON DELETE CASCADE` на `exercises.workout_id` и `cardio_sessions.workout_id` — ручная чистка связанных таблиц не нужна.
4. После удаления — перерисовать `/history` с того же офсета.
5. Повторный тап / запись уже удалена (`error.code` не найдена) — не показывать ошибку, просто обновить список.

## Редактирование

Точечные правки, доступные для конкретной тренировки (набор кнопок собирается динамически из её состава):

- **Дата** — всегда доступна.
- **Разминка** (`warmup_minutes`) — если была указана.
- **Подходы** — отдельная кнопка на каждое упражнение в тренировке; правка — полная замена строки подходов (`40x12, 40x12, 42.5x10`), переиспользуется парсер из `setsInputStep.ts`.
- **Кардио-блок** — если есть: длительность, дистанция, пульс, уклон; переиспользуется парсер из `cardioStep.ts`.

Флоу: тап `✏️` → бот показывает меню из перечисленных выше пунктов → тап на пункт → бот просит новое значение тем же текстом/форматом, что и при создании тренировки → тот же парсер валидирует ввод → при ошибке — то же сообщение об ошибке, что в мастере создания, режим правки не прерывается → при успехе — `UPDATE` нужной таблицы (с проверкой `user_id` через join на `workouts`, если апдейт идёт в `exercises`/`cardio_sessions`) → возврат в `/history` с обновлённой карточкой на том же офсете.

## Технические детали

- Новая мини-FSM по образцу `state.ts`: `Map<userId, EditSession>`, где `EditSession = { workoutId, field, exerciseId? }`. Отдельная от `drafts` (используется для создания тренировки), чтобы не пересекаться по состоянию, если пользователь одновременно начал новую тренировку и правит старую.
- Новые функции:
  - `db/workouts.ts`: `deleteWorkout(userId, workoutId)`, `updateWorkoutDate(userId, workoutId, date)`, `updateWarmup(userId, workoutId, minutes)`
  - `db/exercises.ts`: `updateExerciseSets(userId, exerciseId, sets)` (с join-проверкой владельца через `workouts`)
  - `db/cardio.ts`: `updateCardioSession(userId, cardioSessionId, params)` (аналогично, join-проверка)
- Новый файл `src/handlers/workout/editStep.ts` (или расширение `history.ts`) — обработка `we:`/`wd:` колбэков и текстовых ответов в режиме правки.
- Новые keyboard-функции в `keyboards.ts`: `deleteConfirmKeyboard(workoutId, offset)`, `editMenuKeyboard(workout, exercises, cardio)`.

## Тестирование

Формального набора тестов в проекте нет (как и для остальных фич). Проверка — ручной прогон в реальном боте (dogfood):
1. Создать тестовую тренировку разных типов (силовая, кардио, смешанная).
2. Удалить одну из них через `/history`, убедиться, что подтверждение работает, запись и связанные упражнения/кардио пропали, список перерисовался на том же офсете.
3. Отредактировать дату, разминку, подходы упражнения, параметры кардио — проверить, что новое значение отражается в `/history`, `/progress` и `/export`.
4. Проверить ввод некорректного значения при правке — бот должен показать ошибку и не выходить из режима правки.
5. Проверить повторный тап на уже удалённую/изменённую карточку (двойной тап) — не должно быть краша или дублирующегося запроса.

## Вне области этого спека

- Смена типа тренировки, добавление/удаление отдельных упражнений.
- Заметки к тренировке, учёт личных рекордов, повтор тренировки по шаблону — отдельные последующие спеки.
