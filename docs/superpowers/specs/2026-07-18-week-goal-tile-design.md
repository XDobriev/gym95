# Плитка «цель недели» вместо «объёма за неделю»

## Контекст и цель

В шапке Mini App (`SummaryBar`) три плитки: `🔥 недель подряд` (стрик),
`всего тренировок`, `объём за неделю` (сумма `weight×reps` за текущую неделю в
тоннах, напр. «3.4 т»).

Третья плитка — «объём за неделю» — не работает как витринная метрика:

- **Смешивает несопоставимое.** Тоннаж приседа и бицепс-подъёма складывается в
  одно число, которое физиологически мало что значит.
- **Игнорирует кардио.** `cardio_sessions` в тоннаж не попадают — после чисто
  кардио-недели плитка показывает «0.0 т».
- **Ломается на частичной неделе.** В понедельник почти ноль, к воскресенью —
  максимум; одно и то же усилие выглядит то провалом, то успехом.
- **«3.4 т» — число без контекста.** Не видно, много это или мало, растёт или
  падает. Нет цели, нет тренда — нет мотивации.
- **Не универсально.** Тоннаж — жаргон; для широкой аудитории непонятен.

**Решение.** Заменить третью плитку на **прогресс к недельной цели по частоте
тренировок** — `weekWorkouts / weekGoal`, напр. «2 / 3», с тонким прогресс-баром
и состоянием «цель достигнута». Это:

- мгновенно понятно любому (никакого жаргона);
- мотивирует — незавершённость тянет закрыть неделю (эффект Зейгарник);
- закрывает продуктовое желание «завести цели» — плитка становится лицом этой
  функции;
- дёшево и надёжно: данные `weekWorkouts` уже считаются в `getSummary`, парсинг
  `sets` не нужен;
- хорошо садится в тройку: *стрик* (не рви серию) + *всего тренировок*
  (масштаб пути) + *цель недели* (добей эту неделю) — три разных горизонта без
  дублирования «сколько всего».

**Аудитория и цель пользователя** (из уточнения): планируется расширять на
широкий круг → метрика должна быть понятна без объяснений; пользователь хочет
завести цели по частоте; роль плитки — **мотивировать**. Всё это указывает
именно на прогресс-к-цели.

**Область этого спека:** замена третьей плитки и введение недельной цели как
контракта API (пока — константа на бэкенде). Не входит: экран настроек для
редактирования цели пользователем, перенос тоннажа на экран «Прогресс» — это
последующие итерации (см. «Дальнейшее развитие»).

## Контракт данных

`SummaryResponse` уже отдаёт `weekWorkouts` (число тренировок за текущую
календарную неделю, считается через `mondayKey`). Добавляем одно поле:

```ts
export interface SummaryResponse {
  totalWorkouts: number;
  weekStreak: number;
  weekVolumeKg: number;   // остаётся в ответе — пригодится для экрана «Прогресс»
  weekWorkouts: number;
  weekGoal: number;       // НОВОЕ: цель по числу тренировок в неделю
}
```

Цель **владеет бэкенд** (а не фронт-константа) — чтобы будущий переход на
персональную цель был правкой одной строки в `getSummary`, а не изменением
контракта. Пока значение — константа `WEEKLY_GOAL_DEFAULT = 3`:

```ts
// src/webapp/queries.ts (и зеркало webapp/api/_lib/queries.ts)
const WEEKLY_GOAL_DEFAULT = 3; // недельная цель по частоте; позже — из профиля пользователя
// ...
return { totalWorkouts, weekStreak, weekVolumeKg, weekWorkouts, weekGoal: WEEKLY_GOAL_DEFAULT };
```

**Почему 3.** Разумный дефолт для широкой аудитории (не пугает новичка, не
занижает планку регулярному). Плитка работает сразу, до появления настроек.

**Зачем два файла.** Активный бэкенд на Render — `src/webapp/server.ts` →
`src/webapp/queries.ts`. `webapp/api/_lib/queries.ts` — legacy Vercel-путь
(сейчас не деплоится, но зеркалит контракт). Правим оба, чтобы типы и форма
ответа не разъехались. Типы `SummaryResponse` продублированы в
`src/types/webapp.ts` (бэкенд) и `webapp/shared/types.ts` (фронт) — обновляем оба.

## UI: третья плитка

Вместо `formatVolume(weekVolumeKg)` + подписи «объём за неделю»:

- **Значение:** `{weekWorkouts} / {weekGoal}` — напр. «2 / 3».
- **Подпись:** «цель недели».
- **Прогресс-бар:** тонкая полоса под значением, заполнение
  `min(weekWorkouts / weekGoal, 1)` градиентом `--accent-grad`.
- **Состояние «достигнута»** (`weekWorkouts >= weekGoal`): плитка получает
  класс-модификатор — акцентная заливка (как у hero-плитки стрика) и `✓` рядом
  со значением, полоса залита целиком. Микро-награда за закрытие недели.
- **Перевыполнение** (`weekWorkouts > weekGoal`, напр. «4 / 3»): показываем как
  есть — бар залит на 100 %, состояние «достигнута». Не прячем лишние тренировки.
- **Пустая неделя** (`weekWorkouts = 0`): «0 / 3», пустой бар. Это норма в
  начале недели, а не ошибка — прогресс-бар именно для того и растёт по ходу
  недели (в отличие от тоннажа, здесь частичная неделя — фича, а не баг).

`weekGoal` теоретически может прийти `0` (если позже кто-то так настроит) —
защищаемся: при `weekGoal <= 0` бар считаем пустым (без деления на ноль),
состояние «достигнута» не показываем.

## Файлы к изменению

- `webapp/shared/types.ts` — `+ weekGoal` в `SummaryResponse`.

- `webapp/api/_lib/queries.ts` — то же (зеркало).
- `webapp/src/components/SummaryBar.tsx` — новая третья плитка + прогресс-бар.

- `webapp/src/format.ts` — `formatVolume` остаётся (пригодится на «Прогрессе»),
  из плитки убираем его использование.

## Тестирование

Формальных тестов в проекте нет — ручной прогон (dogfood):

1. Собрать webapp (`npm run build` в `webapp/`) и прогнать `tsc` бэкенда —
   билд зелёный, контракт `SummaryResponse` синхронен на фронте и бэке.
2. Неделя с 0 тренировок → «0 / 3», пустой бар, без состояния «достигнута».
3. 1–2 тренировки → «1 / 3», «2 / 3», бар частично залит.
4. 3 тренировки → «3 / 3», `✓`, акцентная заливка, полный бар.
5. 4+ тренировок → «4 / 3», бар 100 %, состояние «достигнута» (перевыполнение).
6. Проверить читаемость в светлой и тёмной теме Telegram.

## Дальнейшее развитие (вне области)

- **Персональная цель.** Экран/команда настройки: хранить `week_goal` в профиле
  пользователя (новая колонка/таблица), `getSummary` читает её вместо константы.
  Контракт `SummaryResponse.weekGoal` уже готов — фронт менять не нужно.
- **Перенос тоннажа.** «Объём по неделям» как график на экране «Прогресс», где у
  него есть контекст (тренд во времени) — там метрика читается, в отличие от
  одиночной плитки. `weekVolumeKg` для этого остаётся в ответе.
- **Празднование цели.** При достижении `weekGoal` — haptic + короткая анимация
  заливки плитки.
