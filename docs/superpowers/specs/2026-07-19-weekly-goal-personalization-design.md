# Персональная цель недели + график тоннажа

## Контекст и цель

Спек [2026-07-18-week-goal-tile-design.md](2026-07-18-week-goal-tile-design.md) заменил плитку
«объём за неделю» на прогресс к цели `weekWorkouts / weekGoal`, но `weekGoal` там —
захардкоженная константа `WEEKLY_GOAL_DEFAULT = 3` на бэкенде. В разделе «Дальнейшее
развитие» того спека обозначены два независимых следующих шага: сделать цель
персональной (настраиваемой пользователем) и вернуть тоннаж как график на экран
«Прогресс» (где у метрики есть контекст — тренд во времени, а не одинокое число).

Оба технически не связаны (разные запросы и модели данных), но делаются одной
итерацией, так как решены вместе.

**Область этого спека:** настройка недельной цели через `/settings` в боте +
переключаемый график тоннажа по неделям на экране «Прогресс» в Mini App. Не входит:
экран настроек в самом Mini App (его не существует — все настройки только через
бота), celebration-анимация при достижении цели (отдельный будущий пункт).

## 1. Данные и бэкенд

- Миграция (`migrations/schema.sql`):
  ```sql
  alter table user_settings add column if not exists week_goal int not null default 3
    check (week_goal between 1 and 7);
  ```
- `src/db/settings.ts`: `UserSettings` получает `week_goal: number`;
  `DEFAULT_SETTINGS.week_goal = 3`.
- `src/webapp/queries.ts`: `getSummary` читает `week_goal` через `getSettings(userId)`
  вместо константы `WEEKLY_GOAL_DEFAULT` (которая удаляется).
- Новая функция `getVolumeHistory(userId, weeks = 12)`: один запрос
  `exercises.select('sets, workouts!inner(user_id, date)')` с фильтром по дате
  (последние 12 календарных недель), группировка по `mondayKey` (переиспользуется
  существующая функция из этого же файла) в памяти, **zero-fill** недель без
  тренировок, сортировка по возрастанию даты. Тоннаж считается так же, как и раньше
  в `weekVolume` — только `weight × reps` по силовым сетам, кардио не учитывается
  (сохраняем существующее поведение метрики, не меняем её смысл).
- Новый DTO в `src/types/webapp.ts` (и зеркало в `webapp/shared/types.ts` — фронтовый
  контракт зеркалим всегда):
  ```ts
  export interface VolumeHistoryResponse {
    weeks: { weekStart: string; volumeKg: number }[];
  }
  ```
- Новый роут `GET /api/volume-history` в `src/webapp/server.ts`, авторизация — та же
  `authOrThrow` через `X-Telegram-Init-Data`, что и у остальных `GET`-эндпоинтов.
- **Без зеркала в `webapp/api/*`** (legacy Vercel-путь) — по прецеденту фичи
  редактирования/удаления тренировок (`4f58d64`), которая тоже не зеркалилась.
  Legacy-путь остаётся синхронным только по уже существующим до него эндпоинтам.

## 2. `/settings` в боте — задать цель

- `settingsText()` — добавляется строка `Цель недели: N тренировок`.
- `settingsKeyboard()` — новая кнопка `🎯 Изменить цель` → callback `s:goal`.
- Новый `Set<number>` `waitingForGoal` в `src/handlers/settings.ts`, отдельный от
  `waitingForTime` (тот же паттерн мини-состояния, что уже используется для времени
  напоминаний).
- `bot.action('s:goal', ...)`: `waitingForGoal.add(userId)`, ответ — «Сколько
  тренировок в неделю — цель? Число от 1 до 7.»
- В общем `bot.on('text', ...)` роутере: если `waitingForGoal.has(userId)` —
  `parseInt`, валидация `1 <= n <= 7` (целое). При ошибке — сообщение с просьбой
  повторить ввод, режим правки не прерывается (симметрично обработке времени). При
  успехе — `upsertSettings(userId, { week_goal: n })`, подтверждение с
  обновлённой клавиатурой, `waitingForGoal.delete(userId)`.

## 3. Mini App — экран «Прогресс»

- Наверху `ProgressScreen` — переключатель из двух пилюль: «По упражнению» /
  «Общий тоннаж», локальный `useState<'exercise' | 'volume'>`.
- **«По упражнению»** — существующий вид без изменений (селект упражнения + линейный
  график макс. веса + список сессий с PR-подсветкой).
- **«Общий тоннаж»** — новый `BarChart` (Recharts, тот же стиль оформления — glow-фильтр,
  цвета из CSS-переменных `--lime-ink`/`--accent-grad`, что и у линейного графика) по
  данным `getVolumeHistory`: X — подпись недели (`formatDateDDMM(weekStart)`), Y —
  тоннаж, подписи через уже существующий `formatVolume` из `webapp/src/format.ts`
  (тот, что раньше использовался в плитке «объём за неделю»). Пустая история (0
  тренировок за все 12 недель, но `weeks` всё равно приходит с 12 нулевыми точками)
  — показываем плоский график, не `EmptyState` (это осмысленное «пока пусто», а не
  ошибка загрузки). `EmptyState` — только если у пользователя вообще нет ни одной
  тренировки (тот же критерий, что уже используется для вкладки «По упражнению»:
  `names.length === 0` из `/api/exercises`).
- Новый метод в `webapp/src/api.ts`:
  `volumeHistory: () => get<VolumeHistoryResponse>('/api/volume-history')`.

## 4. Обработка ошибок и краевые случаи

- `week_goal` защищён CHECK-constraint в БД (1–7) — дополнительная подстраховка на
  случай прямой записи мимо валидации в боте; сам бот отсекает невалидный ввод раньше.
- `getVolumeHistory` всегда возвращает ровно `weeks` точек (zero-filled), даже если
  тренировок не было вообще — потребителю (фронту) не нужно самому обрабатывать
  «дырки» в датах.
- Существующая защита `weekGoal <= 0` в `SummaryBar.tsx` (см. спек 07-18) не
  трогается — источник значения теперь настоящая пользовательская настройка вместо
  константы, но контракт и защита от деления на ноль те же.

## Тестирование

Формальных тестов в проекте нет — ручной прогон (dogfood), как и для предыдущих фич:

1. `/settings` → «Изменить цель» → ввести 5 → плитка «цель недели» в Mini App
   показывает `X / 5`.
2. Ввести невалидное значение (0, 8, «пять») → бот просит повторить, режим правки
   не выходит.
3. Переключатель на «Прогрессе» → «Общий тоннаж» → бар-чарт за 12 недель, недели
   без тренировок — нулевые столбики, без краша.
4. Аккаунт без единой тренировки → `EmptyState` на вкладке «По упражнению» (как и
   раньше) и на вкладке «Общий тоннаж».
5. `tsc` бэкенда + `npm run build` в `webapp/` — зелёная сборка, контракты
   `VolumeHistoryResponse` синхронны на фронте и бэке.
6. Проверить читаемость бар-чарта в светлой и тёмной теме Telegram.

## Вне области этого спека

- Экран настроек внутри самого Mini App — настройки остаются только через
  `/settings` в боте.
- Celebration-анимация при достижении цели недели (haptic + заливка плитки) —
  отдельный будущий пункт из спека 07-18.
- Зеркалирование `/api/volume-history` в legacy `webapp/api/*` (Vercel-путь).
